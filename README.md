# Лабораторная работа: Основы Android-разработки (или название вашего проекта)

Краткое описание: Приложение, разработанное в ходе выполнения лабораторной работы для изучения жизненного цикла Activity и основ интерфейса Android.

## Цель работы

Изучение жизненного цикла Activity в Android, освоение методов обработки состояний приложения, а также получение навыков логирования событий с помощью класса `Log`.

## Контрольные вопросы

### 1. Какие три метода вызываются при запуске приложения?
При стандартном запуске приложения (переход из состояния "нет процесса" в состояние "работает") последовательно вызываются следующие методы:
1.  `onCreate()` — создание активности.
2.  `onStart()` — запуск активности (она становится видимой).
3.  `onResume()` — возобновление работы (активность получает фокус и готова к взаимодействию с пользователем).

### 2. Чем отличается `onPause()` от `onStop()`?
*   **`onPause()`:** Вызывается, когда активность частично видима, но теряет фокус (например, при появлении диалогового окна или режима "картинка в картинке"). Активность всё ещё находится в памяти и частично видна.
*   **`onStop()`:** Вызывается, когда активность больше не видна пользователю (например, при запуске новой активности или сворачивании приложения в фон). Активность остаётся в памяти, но не отображается на экране.

### 3. Когда вызывается `onRestart()`?
Метод `onRestart()` вызывается перед тем, как остановленная активность (которая была в состоянии `onStop`) снова становится видимой для пользователя. После него всегда вызывается `onStart()`. Типичный сценарий — пользователь вернулся к приложению через экран недавних приложений.

### 4. Что происходит при повороте экрана?
По умолчанию при повороте экрана (изменении конфигурации устройства) текущая активность уничтожается и создается заново. Система вызывает:
*   `onPause()`
*   `onStop()`
*   `onDestroy()`
*   А затем: `onCreate()`, `onStart()`, `onResume()`.

Это делается для того, чтобы приложение могло подгрузить альтернативные ресурсы (например, макеты для ландшафтной ориентации).

### 5. В чём разница между `remember` и `rememberSaveable`?
Обе функции используются в Jetpack Compose для хранения состояния.
*   **`remember`:** Сохраняет значение только во время текущей рекомпозиции. При изменении конфигурации (например, повороте экрана) или уничтожении активности значение сбрасывается.
*   **`rememberSaveable`:** Сохраняет значение не только при рекомпозиции, но и при воссоздании активности (например, после поворота экрана). Похоже на `onSaveInstanceState`, так как использует `Bundle` для сохранения состояния.

### 6. Почему `onCreate()` должен вызывать `super.onCreate()`?
Вызов `super.onCreate(savedInstanceState)` обязателен, так как он выполняет важную работу по инициализации базового класса `Activity` (или `AppCompatActivity`). Этот вызов восстанавливает состояние иерархии представлений, настраивает контекст активности и выполняет другие внутренние операции Android-фреймворка. Если не вызвать родительский метод, приложение вызовет исключение (`SuperNotCalledException`).

### 7. Для чего используется класс `Log`?
Класс `Log` используется для вывода отладочных сообщений в панель **Logcat** в Android Studio. Это помогает разработчику отслеживать выполнение кода, проверять значения переменных и фиксировать события (например, вызов методов жизненного цикла). Основные методы: `Log.v()`, `Log.d()`, `Log.i()`, `Log.w()`, `Log.e()`.

### 8. Какой метод вызывается только один раз за время жизни Activity?
Метод **`onCreate()`**. Он вызывается только один раз за весь жизненный цикл конкретного экземпляра Activity (при его создании).
